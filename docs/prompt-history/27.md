## Postman MCP'sinin Temel Yetenekleri

### 1. **Collection Yönetimi**
- **Oluşturma**: `createCollection` - Postman Collection v2.1.0 formatında koleksiyon oluşturma
- **Okuma**: `getCollection` - Koleksiyon detaylarını görüntüleme (minimal/full model seçenekleri)
- **Listeleme**: `getCollections` - Workspace'teki tüm koleksiyonları listeleme
- **Silme**: `deleteCollection` - Koleksiyon silme
- **Güncelleme**: `patchCollection`, `putCollection` - Koleksiyon güncelleme
- **Çoğaltma**: `duplicateCollection` - Koleksiyonu başka workspace'e kopyalama
- **Tag Yönetimi**: `getCollectionTags`, `updateCollectionTags` - Koleksiyon etiketlerini yönetme

### 2. **Request/Response Yönetimi**
- **Request**: `createCollectionRequest`, `getCollectionRequest`, `updateCollectionRequest`, `deleteCollectionRequest`
- **Response**: `createCollectionResponse`, `getCollectionResponse`, `updateCollectionResponse`, `deleteCollectionResponse`
- **Folder**: `createCollectionFolder`, `getCollectionFolder`, `updateCollectionFolder`, `deleteCollectionFolder`
- Tüm HTTP metodları desteklenir (GET, POST, PUT, DELETE, PATCH, vb.)
- GraphQL, form-data, urlencoded, raw, binary veri modları

### 3. **Collaboration (İş Birliği)**
- **Yorumlar**: Collection, folder, request ve response üzerine yorum ekleme/silme
- **Fork**: `createCollectionFork`, `getCollectionForks` - Koleksiyon fork'lama ve fork'ları görüntüleme
- **Merge**: `mergeCollectionFork` - Fork'ları birleştirme
- **Pull**: `pullCollectionChanges` - Değişiklikleri çekme
- **Thread Desteği**: Yorumlara cevap verme (threadId kullanarak)
- **Kullanıcı Etiketleme**: Yorumlarda @mention desteği

### 4. **Environment Yönetimi**
- `createEnvironment` - Environment oluşturma
- `getEnvironment`, `getEnvironments` - Environment görüntüleme
- `putEnvironment`, `patchEnvironment` - Environment güncelleme
- `deleteEnvironment` - Environment silme
- Secret ve default değişken tipleri
- Workspace bazlı global değişkenler

### 5. **Mock Server**
- `createMock` - Mock server oluşturma
- `getMock`, `getMocks` - Mock server görüntüleme
- `updateMock` - Mock server güncelleme
- `deleteMock` - Mock server silme
- `publishMock`, `unpublishMock` - Yayınlama/yayından kaldırma
- Public/private ayarları

### 6. **Monitor**
- `createMonitor` - API monitör oluşturma
- `getMonitor`, `getMonitors` - Monitör görüntüleme
- `updateMonitor` - Monitör güncelleme
- `deleteMonitor` - Monitör silme
- `runMonitor` - Monitör manuel çalıştırma
- Cron zamanlama, bölge seçimi, bildirim ayarları, retry mekanizması

### 7. **API Specification (Spec Hub)**
- **Desteklenen Formatlar**: OpenAPI 2.0/3.0/3.1, AsyncAPI 2.0, Protobuf 2/3, GraphQL
- `createSpec` - Spec oluşturma (tek veya çok dosyalı)
- `getSpec`, `getAllSpecs` - Spec görüntüleme
- `updateSpecProperties` - Spec güncelleme
- `deleteSpec` - Spec silme
- `createSpecFile`, `getSpecFile`, `updateSpecFile`, `deleteSpecFile` - Dosya işlemleri
- `generateCollection` - Spec'ten collection oluşturma
- `generateSpecFromCollection` - Collection'dan spec oluşturma
- `syncCollectionWithSpec`, `syncSpecWithCollection` - Senkronizasyon

### 8. **Workspace Yönetimi**
- `createWorkspace` - Workspace oluşturma
- `getWorkspace`, `getWorkspaces` - Workspace görüntüleme
- `updateWorkspace` - Workspace güncelleme
- `deleteWorkspace` - Workspace silme
- Workspace tipleri: personal, private, public, team, partner
- `getWorkspaceGlobalVariables`, `updateWorkspaceGlobalVariables` - Global değişkenler

### 9. **Analytics**
- `getAnalyticsData` - Detaylı analitik veriler
- `getAnalyticsMetadata` - Mevcut metrikler kataloğu
- **Resource Tipleri**: user, team, workspace
- **Metrikler**: API çağrıları, aktif kullanıcılar, başarı oranları, workspace dağılımı, lisans tüketimi vs.
- Filtreleme: duration, workspace tipi, response status

### 10. **Private API Network**
- `addWorkspaceToPrivateNetwork` - Workspace'i private network'e ekleme
- `removeWorkspaceFromPrivateNetwork` - Çıkarma
- `listPrivateNetworkWorkspaces` - Private network workspace'lerini listeleme
- `searchPostmanElementsInPrivateNetwork` - Private network'te arama

### 11. **Kullanıcı ve Kimlik**
- `getAuthenticatedUser` - Mevcut kullanıcı bilgilerini alma
- Kullanıcı ID, username, teamId, rol bilgileri

### 12. **Collection Runner**
- `runCollection` - Collection'ı çalıştırma
- Test sonuçlarını alma
- Environment ve data file desteği

### 13. **Documentation**
- `publishDocumentation` - Dokümantasyon yayınlama
- `unpublishDocumentation` - Yayından kaldırma

### 14. **Arama**
- `searchPostmanElementsInPublicNetwork` - Public network'te arama
- `searchPostmanElementsInPrivateNetwork` - Private network'te arama
- Collection, workspace, API arama

### 15. **Kod Üretimi**
- `getCodeGenerationInstructions` - API'den kod üretme talimatları
- API client kodu oluşturma rehberi

### 16. **Authentication Desteği**
- Basic Auth, Bearer Token, API Key, OAuth1/2
- AWS Signature (awsv4), Hawk, NTLM, Digest
- Akamai Edgegrid, JWT, ASAP
- No Auth seçeneği

### 17. **Transfer & Organization**
- `transferCollectionFolders` - Folder'ları taşıma
- `transferCollectionRequests` - Request'leri taşıma
- `transferCollectionResponses` - Response'ları taşıma

## Kullanım Senaryoları

1. **API Geliştirme**: Collection oluştur → Request ekle → Test yaz → Environment tanımla
2. **Test Otomasyonu**: Monitor kur → Zamanlama ayarla → Bildirim yapılandır
3. **Mock Geliştirme**: Mock server oluştur → Collection'a bağla → Test et
4. **Dokümantasyon**: Spec oluştur → Collection'a sync et → Yayınla
5. **Takım Çalışması**: Collection fork'la → Değişiklik yap → Merge et → Yorum ekle
6. **Analitik**: API kullanımlarını izle → Hata oranlarını görüntüle → Performans takibi
