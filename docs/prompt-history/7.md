## Plan: Product-Service Veritabanı Implementasyonu

**TL;DR:** H2 In-Memory DB ile JPA katmanı ekleniyor. Entity + Repository + 2 ayrı Service (V1/V2) + Exception handling yeni katmanlar oluştururken, 2 controller ve 2 config dosyası güncelleniyor.

---

**Faz 1 — Dependency & Konfigürasyon** *(2 dosya)*
1. `pom.xml` → `spring-boot-starter-data-jpa` ve `com.h2database:h2` eklenir.
2. application.yml → H2 datasource, JPA (`ddl-auto: create-drop`, `show-sql`), H2 console config eklenir.

**Faz 2 — Domain Katmanı** *(2 yeni dosya)*

3. `entity/Product.java` → `@Entity` sınıfı: `id (UUID, @GeneratedValue)`, `name`, `description (nullable)`, `price`, `stock`, `sku (@Column(unique=true))`.
4. `repository/ProductRepository.java` → `JpaRepository<Product, UUID>` + `Optional<Product> findBySku(String sku)`, `boolean existsBySku(String sku)`, arama için `Page<Product> findByNameContainingIgnoreCaseOrSkuContainingIgnoreCase(...)`.

**Faz 3 — Exception Katmanı** *(2 yeni + 1 değişiklik)*

5. `exception/ProductNotFoundException.java` → `RuntimeException`, `"PRODUCT_NOT_FOUND"`.
6. `exception/SkuAlreadyExistsException.java` → `RuntimeException`, `"SKU_CONFLICT"`.
7. `exception/GlobalExceptionHandler.java` → 404 (`ProductNotFoundException`) ve 409 (`SkuAlreadyExistsException`) handler'ları eklenir.

**Faz 4 — Service Katmanı** *(2 yeni dosya)*

8. `service/ProductServiceV1.java` → `@Service`, `ProductRepository` inject, `list / create / getById / replace / delete / updateStock` metodları; entity ↔ `ProductResponse` mapping.
9. `service/ProductServiceV2.java` → `@Service`, aynı operasyonlar; entity ↔ `ProductResponseV2` mapping (`description` dahil).

**Faz 5 — Controller Güncellemesi** *(2 değişiklik)*

10. `ProductController.java` → `ProductServiceV1` inject, TODO yorum satırları kaldırılır, servis çağrıları yazılır.
11. `ProductControllerV2.java` → `ProductServiceV2` inject, aynı şekilde.
